### Event Objectives

The main purpose of this event was to bring the most important announcements and architectural ideas from AWS re:Invent closer to the local tech community in Ho Chi Minh City. As an attendee, I expected to update my knowledge about the newest AWS services, observe how AWS is solving real-world infrastructure problems, and understand how modern cloud architecture is evolving toward better performance, lower cost, and stronger security.

This event was especially valuable because it was not limited to marketing-level announcements. Instead, it focused on practical infrastructure modernization topics such as storage optimization, compute efficiency, database cost management, edge security, and better connectivity for Vietnamese users.

### Main Content Highlights

#### App-DB Modernization Workshop
The workshop showed how modern application and database systems can be redesigned from the ground up instead of being patched piece by piece. The speakers emphasized a mindset shift: instead of keeping everything in a single monolithic structure, teams should think in terms of smaller services, clearer ownership, and more flexible deployment patterns.

The most useful ideas for me were the combination of modern application decomposition, event-driven design, and serverless execution. I also learned how AI-assisted development tools such as Amazon Q Developer can help engineers write code faster, detect issues earlier, and reduce repetitive work during implementation.

#### Storage and Data
One of the most memorable parts of the recap was the storage section. I learned that AWS is continuously improving the way data is stored, accessed, and optimized. Features such as intelligent tiering and vector-enabled search show that storage is no longer just about keeping files in one place; it is now tightly connected with performance, analytics, and AI workloads.

I also noticed the practical importance of easier access management. Using tags to manage S3 permissions is a meaningful improvement when multiple teams work on the same data platform, because it simplifies governance and reduces manual overhead.

#### Servers and Running Applications
The compute section showed that AWS is still pushing performance and operational simplicity forward. The introduction of new Graviton processors demonstrates how cloud platforms keep improving price-performance efficiency. I also found it interesting that AWS Backup support for EKS reduces the need for custom scripts and gives teams a cleaner recovery strategy.

Another strong message from this part of the event was reliability. Cloud systems must stay available even when errors happen, and the platform features presented during the recap reflected that principle clearly.

#### Networking and Security
The networking and security updates were also very practical. I learned that protection at the edge is becoming more important than ever, especially for modern applications with global users and sensitive workloads. CloudFront security enhancements, quantum-resistant protection concepts, and fixed-price options all showed that the cloud is moving toward a more stable, predictable, and secure operating model.

#### Databases
The database section reminded me that cost optimization and data access speed must go together. AWS highlighted savings for services such as RDS and DynamoDB when workloads are used consistently, while OpenSearch improvements showed that search can now become smarter and more natural-language-friendly through GPU support.

### What I Learned

From this event, I gained a broader understanding of how AWS is shaping the future of cloud infrastructure. I learned that:

- modernization is not only about replacing old systems, but about redesigning them for flexibility;
- storage choices must be aligned with cost, access patterns, and AI use cases;
- compute platforms should be evaluated not only by speed, but also by operational simplicity and recovery capability;
- security is increasingly tied to edge architecture and platform defaults;
- database and search platforms are becoming more intelligent and workload-aware.

### Lessons Learned

The biggest lesson I took away is that cloud architecture is no longer just about “running servers online.” It is about building systems that are resilient, cost-aware, secure, and ready for rapid change. I also realized that the best engineers are not the ones who memorize services, but the ones who understand how each service contributes to an overall system strategy.
